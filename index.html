BankApp – DevOps Pipeline

This project involved designing and deploying a secure and scalable CI/CD pipeline for a microservices-based banking platform. Jenkins was used as the automation server, integrated with Docker and Helm to manage containerized deployments. Each service was built as a separate container image and deployed to AWS EKS using Helm charts with rollback strategies enabled via Helm hooks. Infrastructure was defined using Terraform modules, which created AWS resources such as EKS clusters, IAM roles, and security groups. Kubernetes Horizontal Pod Autoscalers (HPA) were implemented to automatically scale services based on resource usage metrics. Observability was enhanced using Amazon CloudWatch for logs and metrics, while Grafana provided real-time dashboards. This setup improved release velocity and reduced deployment errors, enhancing operational confidence in production environments.

DevOps Pipeline for MERN-based Crypto Application

Developed a CI/CD pipeline for a cryptocurrency tracking application built with the MERN stack (MongoDB, Express.js, React, Node.js). The pipeline used GitHub Actions to automate testing, building, and deployment of Dockerized microservices to AWS EKS. Terraform was used to provision infrastructure including VPCs, subnets, EKS clusters, and RDS. Ansible playbooks handled EC2 configuration, ensuring secure and repeatable setups across environments. Prometheus and Grafana were configured for monitoring application health and infrastructure metrics. Fluent Bit shipped container logs to AWS CloudWatch, enabling centralized log aggregation and easier debugging. This pipeline achieved faster feature delivery, improved reliability, and met strict compliance and availability requirements.

Multi-Environment Terraform & Ansible

This infrastructure-as-code project focused on building a robust multi-environment (Dev, Staging, Prod) deployment architecture. Using Terraform, I created reusable modules to provision EC2 instances, S3 buckets, and DynamoDB tables with environment-specific configurations. DevOps Pipeline for MERN E-commerce App

Led the end-to-end DevOps implementation for a MERN-based e-commerce platform. Jenkins pipelines were set up to automate testing, Docker builds, and deployment to AWS EKS clusters using Helm charts. Terraform provisioned the cloud infrastructure including VPC, EKS, RDS, and IAM resources. Ansible automated the software configuration and environment bootstrapping on EC2 instances, ensuring consistent system states. Real-time observability was implemented using Fluent Bit for log forwarding to CloudWatch and Prometheus/Grafana for metric visualization and alerting. AWS GuardDuty was enabled to monitor malicious activity and harden security posture. The pipeline supported blue/green deployments and zero-downtime updates, improving developer productivity and user experience.

Real-Time Log Analysis using AWS Kinesis

Created a real-time data pipeline to analyze application logs for anomalies, usage trends, and potential security threats. AWS Kinesis Data Streams was used for high-throughput ingestion of log events from multiple microservices and environments. Lambda functions processed the streaming data to filter and transform logs before storing them in Amazon Elasticsearch (OpenSearch). Kibana dashboards visualized metrics like request latency, error spikes, and API usage patterns in real time. CloudWatch Alarms and custom metrics triggered alerts via SNS for critical thresholds, such as sudden traffic surges or suspicious login activity. This setup enabled proactive monitoring, reduced mean-time-to-detection (MTTD), and enhanced situational awareness for operations and security teams.

Lambda Cost Optimizer

Developed a fully serverless Python-based solution to automatically optimize AWS resource usage and reduce costs. A scheduled AWS Lambda function, triggered via CloudWatch Events, performed periodic inventories of EC2 instances, EBS volumes, and Elastic IPs. The function identified stale or unused resources — such as stopped EC2 instances, orphaned EBS volumes, and unattached Elastic IPs — and removed them safely. Stakeholders were notified in real-time via Amazon SNS alerts when cleanup actions were performed. Additionally, the system generated weekly summary reports highlighting reclaimed resources, estimated cost savings, and optimization trends. This proactive approach improved operational efficiency, reduced cloud spend, and provided full transparency into resource lifecycle management.
